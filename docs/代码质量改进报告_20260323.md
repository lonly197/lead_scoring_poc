## 二、P0 级别修复

### 2.1 fillna 顺序错误

**问题位置**：`src/data/loader.py:213, 244`

**问题描述**：
```python
# 错误写法
df[col] = df[col].astype(str).fillna("missing")
```
`astype(str)` 会将 `NaN` 转换为字符串 `"nan"`，后续 `fillna("missing")` 不再生效。

**修复方案**：
```python
# 正确写法
df[col] = df[col].fillna("missing").astype(str)
```

**影响**：缺失值无法正确标记为 "missing"，模型可能学习到错误的特征值。

### 2.2 positive_class 硬编码

**问题位置**：`src/models/predictor.py:152-154`

**问题描述**：
```python
# 错误写法
return proba.iloc[:, 1].values
```
硬编码假设正类概率在 `iloc[:, 1]`，依赖 AutoGluon 内部列顺序。

**修复方案**：
```python
# 正确写法
positive_class = self._predictor.positive_class
return proba[positive_class].values
```

**影响**：AutoGluon 版本升级或不同问题类型可能导致返回错误的概率。

## 三、P1 级别修复

### 3.1 列名匹配逻辑错误

**问题位置**：`src/data/loader.py:81-90`

**问题描述**：
```python
# 错误写法
target_cols = [col for col in df.columns if col.startswith("label_")]
if "clue_level" in df.columns:
    ...
```
实际数据中目标变量命名为 `到店标签_14天`、`线索评级_试驾前`，而非 `label_`、`clue_level`。

**修复方案**：
```python
# 正确写法
target_cols = [col for col in df.columns if "标签" in col or "评级" in col]
if "线索评级_试驾前" in df.columns:
    ...
```

**影响**：统计信息无法正确显示目标变量分布。

### 3.2 默认 ID 列名错误

**问题位置**：`scripts/generate_topk.py:70`

**问题描述**：
```python
# 错误写法
default="门店线索编号"
```
实际数据中的 ID 列名为 `线索唯一ID`。

**修复方案**：
```python
# 正确写法
default="线索唯一ID"
```

**影响**：不显式指定 `--id-column` 时会生成错误的 ID 列名。

## 四、P2 级别优化

### 4.1 类别不平衡处理

**新增功能**：`LeadScoringPredictor` 支持 `sample_weight` 参数

**使用方式**：
```python
predictor = LeadScoringPredictor(
    label="线索评级_试驾前",
    sample_weight="balance_weight",  # 自动平衡类别权重
    weight_evaluation=True,
)
```

**效果**：H 类（17%）和 O 类（1.9%）的召回率提升。

### 4.2 移除双重预处理

**问题**：原 `FeatureEngineer` 手动进行类别编码和缺失值填充，与 AutoGluon 内置预处理冲突。

**解决方案**：
| 功能 | 原处理 | 现处理 |
|------|--------|--------|
| 类别编码 | 手动 `_encode_categories` | AutoGluon `CategoryFeatureGenerator` |
| 缺失值填充 | 手动 `_handle_missing_values` | AutoGluon `FillNaFeatureGenerator` |
| 时间特征提取 | 手动 `_extract_time_features` | 保留（AutoGluon 不自动处理） |

**接口变更**：
```python
# 旧接口
feature_engineer = FeatureEngineer(
    time_columns=[...],
    categorical_columns=[...],
    numeric_columns=[...],
)
df_processed, metadata = feature_engineer.process(df, fit=True, category_mappings=None)

# 新接口
feature_engineer = FeatureEngineer(
    time_columns=[...],
    numeric_columns=[...],  # 可选
)
df_processed, metadata = feature_engineer.process(df)
```

### 4.3 OOT 时间切分验证

**新增功能**：`split_data_oot()` 函数

**使用方式**：
```python
from src.data.loader import split_data_oot

train_df, test_df = split_data_oot(
    df=df,
    target_label="到店标签_14天",
    time_column="线索创建时间",
    cutoff_date="2026-02-01",
)
```

**优势**：更好地模拟真实预测场景（用历史数据预测未来）。

## 五、AutoGluon 兼容性分析

### 5.1 AutoGluon 内置预处理

```
Stage 1: AsTypeFeatureGenerator     → 类型推断与转换
Stage 2: FillNaFeatureGenerator     → 缺失值填充
Stage 3: CategoryFeatureGenerator   → 类别特征处理
Stage 4: DropUniqueFeatureGenerator → 删除单一值特征
Stage 5: DropDuplicatesFeatureGenerator → 删除重复特征
```

### 5.2 兼容性原则

| 手动预处理 | 兼容性风险 | 建议 |
|------------|------------|------|
| 类别编码 | ⚠️ 双重编码风险 | ❌ 不做，交给 AutoGluon |
| 缺失值填充 | ⚠️ 覆盖风险 | ❌ 不做，交给 AutoGluon |
| 异常值移除 | ⚠️ 信息丢失 | ❌ 不做，AutoGluon 自动处理偏斜分布 |
| 时间特征提取 | ✅ 无冲突 | ✅ 可以做 |
| 数值类型转换 | ✅ 无冲突 | ✅ 可以做 |

### 5.3 类别不平衡处理

| 方案 | 兼容性 | 推荐 |
|------|--------|------|
| `sample_weight='balance_weight'` | ✅ 完全兼容 | ✅ 推荐 |
| SMOTE 过采样 | ⚠️ 可能过拟合 | ⚠️ 谨慎使用 |
| 自定义权重列 | ✅ 完全兼容 | ✅ 可选 |
