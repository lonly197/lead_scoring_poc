## 一、问题背景

### 1.1 当前设计存在的问题

| 问题 | 说明 | 影响 |
|------|------|------|
| **目标变量错误** | 使用 `线索评级_试驾前`（销售顾问人工评级）作为预测目标 | 模型学习的是"模仿人类判断"，而非预测真实用户行为 |
| **无法验证业务效果** | 无法验证"H桶试驾率"等业务指标 | POC效果无法与业务KPI对齐 |
| **文档不一致** | README.md 声明目标是"到店预测"，但代码使用"人工评级" | 误导业务方和技术团队 |

### 1.2 根本原因

```
正确设计: 特征 → 预测用户行为（试驾/到店/下订时间）
当前设计: 特征 → 模仿人工评级（销售顾问主观判断）
```

`线索评级_试驾前` 是销售顾问在DMS系统中填写的主观评级，存在：
- **主观偏差**：不同顾问标准不一致
- **信息不全**：顾问可能缺乏某些关键信息
- **历史局限**：基于当时信息判断，可能与实际结果不符

## 二、双轨设计方案

### 2.1 设计架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                         双轨设计方案                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────┐      ┌──────────────┐      ┌──────────────┐     │
│   │  预测模型    │  →   │  概率→等级   │  →   │  可解释输出   │     │
│   │ (行为预测)   │      │  映射层      │      │  (SHAP归因)  │     │
│   └──────────────┘      └──────────────┘      └──────────────┘     │
│                                                                     │
│   预测: 试驾/到店概率      映射: 概率阈值→OHAB      输出: 特征贡献   │
│   目标: 真实用户行为      满足: 业务展示需求       满足: 可解释性   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 三层设计

| 层级 | 职责 | 设计要点 |
|------|------|----------|
| **预测层** | 预测用户真实行为概率 | 使用 `试驾标签_14天` 或 `到店标签_14天` 作为目标 |
| **映射层** | 概率 → OHAB等级 | 基于业务规则的概率阈值映射 |
| **解释层** | 特征归因 | SHAP值 + 业务描述 |

## 四、映射层设计

### 4.1 概率 → OHAB 映射规则

根据特征体系文档的评分阈值：

```python
def probability_to_ohab(prob_7day, prob_14day, prob_21day):
    """
    映射逻辑：
    - H级（75-89分对应）：7天概率≥60% 或 14天概率≥75%
    - A级（60-74分对应）：14天概率≥50% 或 21天概率≥70%
    - B级（45-59分对应）：21天概率≥40%
    - C级（<45分对应）：各时间窗口概率都低
    """
    if prob_7day >= 0.60 or prob_14day >= 0.75:
        return "H"
    elif prob_14day >= 0.50 or prob_21day >= 0.70:
        return "A"
    elif prob_21day >= 0.40:
        return "B"
    else:
        return "C"
```

### 4.2 阈值调优

阈值可根据业务效果动态调整：

| 场景 | 调整动作 |
|------|----------|
| H桶过大且转化下降 | 上调H阈值（如0.60→0.65） |
| H桶过小且漏单 | 下调H阈值（如0.60→0.55） |
| A桶长期不转化 | 缩短A→C的降级周期 |

## 七、验证方案

## 八、向业务方展示方案

### 8.1 POC结果展示

**核心信息**：

1. **模型预测的是"用户会不会在N天内试驾/到店"**
   - 这是客观行为预测，而非主观判断复制

2. **OHAB等级是预测概率的业务映射**
   - H级 = 高概率7天内试驾（≥60%）
   - A级 = 中高概率14天内试驾
   - B级 = 低概率但有意向
   - C级 = 培育池

3. **可解释性输出回答"为什么"**
   - 哪些特征提升了评级？
   - 哪些特征降低了评级？
   - 应该如何跟进？

### 8.2 业务话术示例

> "模型预测这位客户在7天内试驾的概率是68%，所以评定为H级。主要原因是他：
> 1. 首次响应很快（30分钟内）
> 2. 通话时长较长（深度沟通）
> 3. 提到了价格关键词（进入比价阶段）
>
> 建议您在24小时内电话确认试驾时间。"

### 8.3 与人工评级对比

| 维度 | 人工评级 | AI评级 |
|------|----------|--------|
| **判断依据** | 销售顾问主观经验 | 多维度特征综合计算 |
| **一致性** | 不同顾问标准不一 | 统一标准，可追溯 |
| **可解释** | 难以量化归因 | 明确特征贡献 |
| **可优化** | 依赖培训 | 数据驱动迭代 |
